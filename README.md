# نظام إدارة شركة نقل المقاولين 🚛

نظام Web Application يعمل **Mobile First** (يشبه تطبيق الموبايل) لإدارة شركة وسيطة بين المقاولين والسواقين، مع إدارة كاملة للماليات والخزنة والأرباح وتوزيعها على الشركاء.

## التقنيات
- **Next.js 15** (App Router) + **TypeScript**
- **Tailwind CSS** + مكوّنات على نمط **shadcn/ui** + Radix UI
- **Prisma ORM** + قاعدة بيانات (SQLite محليًا، قابلة للتحويل إلى PostgreSQL/Supabase)
- **Server Actions** لكل العمليات
- **PWA** (قابل للتثبيت كتطبيق على الهاتف)
- تكامل **WhatsApp** (Click-to-Chat) و **Telegram Bot**
- واجهة **عربية RTL** + **Dark Mode**

---

## التشغيل المحلي

```bash
npm install          # تثبيت الحزم (تم)
npm run db:push      # إنشاء قاعدة البيانات (تم)
npm run db:seed      # تسجيل رأس المال 200,000 ج.م (تم)
npm run dev          # تشغيل بيئة التطوير
# أو للإنتاج:
npm run build && npm run start
```

ثم افتح: http://localhost:3000

**كلمة مرور الدخول الافتراضية:** `admin123` (غيّرها من `ADMIN_PASSWORD` في `.env`)

## الإعدادات (`.env`)

| المتغير | الوصف |
|---|---|
| `DATABASE_URL` | مسار قاعدة البيانات (SQLite افتراضيًا) |
| `ADMIN_PASSWORD` | كلمة مرور المدير |
| `SESSION_SECRET` | سر توقيع الجلسة — **غيّره لقيمة طويلة عشوائية** |
| `INITIAL_CAPITAL` | رأس المال الابتدائي (200000) |
| `TELEGRAM_BOT_TOKEN` / `TELEGRAM_CHAT_ID` | لإشعارات تيليجرام (اختياري) |
| `CRON_SECRET` | سر حماية مسار الإشعارات المجدولة |
| `COMPANY_WHATSAPP` | رقم واتساب الشركة |

## الوحدات المنجزة

- **لوحة التحكم**: الخزنة، أرباح اليوم/الأسبوع/الشهر، رحلات اليوم/الغد، قيد التنفيذ، العملاء المتأخرون، السواقون المستحق لهم.
- **الطلبات (الرحلات)**: إنشاء (مع إضافة مقاول/سواق فوريًا)، قائمة بفلاتر وبحث، تفاصيل مع كل العمليات (تأكيد/بدء/إنهاء/إلغاء، تحصيل، سداد سواق، ملاحظة)، أزرار واتساب جاهزة.
- **المقاولين**: قائمة، بحث، ملف (المطلوب/المحصّل/الآجل/الأرباح/سجل المدفوعات)، PDF.
- **السواقين**: قائمة، ملف (المستحق/المدفوع/المتبقي)، سداد مستحقات (توزيع تلقائي على الرحلات)، PDF.
- **الشركاء**: نِسب، نصيب كل شريك، السحوبات، الرصيد، تقرير أسبوعي/شهري، **تصفية الخزنة وتوزيع الأرباح**.
- **الماليات**: الخزنة مقسّمة (كاش/إنستا باي/محفظة/فيزا)، كل المؤشرات، المصروفات، **دفتر الأستاذ الموحد**.
- **التقارير**: تقرير مالي إجمالي + تقرير مخصص بين تاريخين، كلها قابلة للطباعة PDF.
- **سجل العمليات (Audit Log)** لكل حركة.

## قواعد العمل المطبّقة
1. لا يمكن إنهاء الرحلة بدون سواق.
2. لا يُدفع للسواق أكثر من المتبقي له.
3. لا يُحصَّل أكثر من قيمة الرحلة.
4. الربح = سعر المقاول − مستحق السواق (والصافي بعد خصم المصروفات).
5. كل الحركات المالية في **دفتر أستاذ واحد**، ومنه تُحسب الخزنة والأرباح.
6. لا يُصرف مبلغ أكبر من رصيد الخزنة في طريقة الدفع.
7. كل العمليات تُسجَّل في Audit Log.

> 💡 **المبالغ تُخزَّن بالقروش (عدد صحيح = الجنيه × 100)** لتفادي أخطاء الكسور العشرية.

## واتساب
الأرقام المصرية تُحفظ بصيغة `01xxxxxxxxx` وتُحوَّل تلقائيًا إلى `20xxxxxxxxxx` لروابط `wa.me`.
- رسالة المقاول: التاريخ/الوقت/المسار/اسم ورقم السواق — **بدون أي معلومات مالية**.
- رسالة السواق: نفس التفاصيل + اسم ورقم المقاول — **بدون سعر المقاول إطلاقًا**.

## إشعارات Telegram (اختياري)
1. أنشئ بوت عبر [@BotFather](https://t.me/BotFather) واحصل على `TELEGRAM_BOT_TOKEN`.
2. احصل على `TELEGRAM_CHAT_ID` (راسل البوت ثم افتح `https://api.telegram.org/bot<token>/getUpdates`).
3. ضع القيمتين في `.env`.
4. اربط خدمة Cron خارجية (مثل [cron-job.org](https://cron-job.org) أو Vercel Cron):
   - **يوميًا**: `GET /api/cron?type=daily&secret=<CRON_SECRET>` → تذكير برحلات الغد.
   - **كل جمعة 12:00 ص**: `GET /api/cron?type=weekly&secret=<CRON_SECRET>` → تذكير تصفية الخزنة + التقرير الأسبوعي.

## النشر على Supabase + Vercel

المخطط مضبوط على **PostgreSQL**. الخطوات:

### 1) قاعدة البيانات على Supabase
1. أنشئ مشروعًا على [supabase.com](https://supabase.com).
2. افتح **SQL Editor** → New query → الصق محتوى ملف [`supabase/schema.sql`](supabase/schema.sql) كاملًا → **Run**.
   - هذا يُنشئ كل الجداول ويُسجّل **رأس المال 200,000 ج.م** تلقائيًا.
3. من **Project Settings → Database → Connection string** انسخ:
   - **Transaction (pooler, 6543)** → `DATABASE_URL` (أضف `?pgbouncer=true`).
   - **Direct (5432)** → `DIRECT_URL`.

### 2) النشر على Vercel
1. اربط مستودع GitHub بمشروع جديد على [vercel.com](https://vercel.com).
2. في **Environment Variables** أضِف كل متغيرات [`.env.example`](.env.example):
   `DATABASE_URL`، `DIRECT_URL`، `ADMIN_PASSWORD`، `SESSION_SECRET`، `CRON_SECRET`، (واختياريًا) متغيرات Telegram.
3. اضغط **Deploy**. أمر البناء `prisma generate && next build` يعمل تلقائيًا.

### تشغيل محلي مع Postgres
انسخ `.env.example` إلى `.env`، املأ `DATABASE_URL`/`DIRECT_URL` بقيم Supabase، ثم:
```bash
npm run db:push   # أو شغّل supabase/schema.sql مرة واحدة
npm run dev
```

> ملاحظة: كل القيم المعرّفة كحالات (status, method...) مخزّنة كنصوص لتوافق كامل.
